    Mobile app design according to specifications

    In this lesson, you will walk through the entire process of designing a mobile app to specified specifications, step-by-step.

    1. Read the following problem and specfication.

    Problem:
      Paul is the Program Director at the ABC University Graduate School of Education. The Graduate School of Education is planning a three-day conference this fall. Until last year, conference information was published on the graduate school website, but he received feedback that there were various inconveniences during the actual conference. He received feedback that it is difficult to check the session information, it is not easy to find the session location, and it is not easy to check the sudden session change. Based on this feedback, Paul decided to create a mobile conferencing app for this year's conference.

    Specification:
      Paul requested to design a mobile conference app containing the following features.
      • Session information: title, date, speaker, and location
      • Detailed session page
      • Speakers information
      • Map information

    4. Design how to transit between different pages.

    Good Job!
    You are alomost done. Design how to move between different pages. Follow the following steps.

    Transition:

    There are Schedule and Session Detail pages.

    We can use CLICK funtion to move to each other.

    When clicking the session item at session list, move to the detaild page. Conversely, when clicking the back icon at the detailed page, go back to the schedule page.

    Transition:

    There are Schedule and Speaker List pages.

    But, there is no item to connect to the speaker list at Schedule page. So, we need another item. We can use TAB icon to move to each other.

    When clicking the Speakers icon at session list, move to the Speaker List page. Conversely, when clicking the back icon at the detailed page, go back to the schedule page.
